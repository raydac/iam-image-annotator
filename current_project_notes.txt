## 2. Architecture & Technical Discoveries
- Java 21 Maven project; use `mvn22` for builds and tests.
- Main app module is `iam-image-annotator-app`; Windows launcher is only included by the `publish` Maven profile.
- PNG annotation persistence is handled through custom PNG chunks and JSON serialization.
- JPEG/JPG files are input-only for editing; annotations are persisted by saving to the same stem with `.png`.
- Visible annotated PNG IDAT is allowed to be lossy for compression, but `iBSE` must preserve the original base image.
- Annotated PNG open speed depends mostly on decoding `iBSE`; current reader avoids allocating/copying preview `IDAT` chunks for files saved by the app.
- Annotation JSON supports `visible`; missing values default to visible for backward compatibility.
- Magic wand flood-fill masks are converted to polygon annotations in `MagicWandMaskContour`; border-edge tracing is used to avoid cycles from irregular boundary-pixel ordering.
- `MainFrame` runs long open/save tasks through a `SwingWorker` plus reusable `ProgressGlassPane`; existing-open tabs are selected immediately.
- `FileTreePanel.refresh()` rebuilds from disk while restoring expanded folder paths and still-existing selections.
- Annotated-image exporter API lives in `exporters.api`; shared exporter helpers live in `exporters.common`; each concrete exporter keeps its implementation classes in its own `exporters.<name>` subpackage.
- YOLO bounding exporter skips plain/unannotated PNGs, writes Ultralytics layout `images/train`, `images/val`, `labels/train`, `labels/val`, exports visible annotations as bounding boxes, writes base images from `iBSE` as JPEG quality 95, writes `data.yaml`, and uses `YoloDatasetSplitter` to keep similar images in the same train/val split.
- YOLO validation split is target-capped at 20% of exported annotated images; oversized visually similar clusters stay together and may make validation smaller rather than overfilling it.
- YOLO class IDs are built from normalized export class names; source annotation labels are not rewritten.
- `AnnotationEntry.id()` is the exported class label and is not unique; use `AnnotationEntry.key()` for selection, mutation, layer ordering, locking, visibility, and UI hit-test identity.
